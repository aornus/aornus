楔子

今天(2022-10-24)把浏览器从谷歌迁移到的微软的Edge浏览器，在迁移密码的过程中，发现我总共300多个账号，却只用了9个密码。甚是惊讶。刚好想起来了上上个月写的半篇文章，现在把它补全电子化，贴到博客上。

为了防止歧路亡羊，建议看到一个觉得很适合自己的方法，就立马停止阅读此文，开始自己设计。

引言

现在(二十一世纪二十年代)很多媒体都在强调个人信息的重要性，强调密码的强度要高，提示说不要使用身份证、生日等容易破解的密码。但却并没有人教我们怎样设计一套自己的密码。

一直以来，人们都在最容易破解的密码之中徘徊。笔者曾经也是如此，四年级第一次去网吧申请的4399账号时随便打的718293竟一度成了后来的支付密码，手机号，姓名拼写等也都无脑的用来用去。可能是幸存者偏差的原因，所有的账户竟然一直没有被盗过号，也一直安心的用着，没有构想更安全的密码。直到2019年高考报名时，班主任一再强调，“报名系统的密码要自己记住，强度要高，绝对不能让别人知道，以免别人篡改你的志愿”。对于这关乎一辈子命运的大事，笔者当然十分上心了。当时高考报名系统的密码要求是8位以上，包含大小写、数字以及符号，笔者一看，平常的垃圾密码肯定是用不成了，便苦思冥想了好几天，终于在复习化学时候，从元素周期表中找到了灵感——利用化学元素与名字的谐音(具体见下文)。当时设计完那个密码，真是提笔而立，为之踌躇满志，庖丁解牛后的感觉也大概是那样了。从那之后，慢慢掌握了一些密码设计的诀窍，逐渐地形成了一个个小窍门，而后这一系列的感兴终于在2022年末形成了文字。

密码的基本要求

本文主要是为了帮助普通人设计一套自己的密码体系，来更好、更方便地生活。本文所指代的密码就是一般的账户密码，而其他譬如凯撒密码、恩尼格码(摁你个妈)机、洛伦兹机、量子加密等加密体系由于极其难认，故此类密码学中的加密方法并不在本文的主要讨论范围内。为避免歧义，笔者决定将这种较短的、常用的、私人性质的密码称为——密符。(灵感来自周朝姜子牙用自己的鱼竿制作的密码——阴符)

笔者认为，这样一套密码体系有应该有两个最基本特点:强私人性与易记性。私人性是首当其冲考虑的，因为每个人的经历不可能完全相同，不具有可重复性。哪怕你偶然读来一句诗，心念一动，根据物理上的平行宇宙假说，你就是进入了另一个平行宇宙了。可以说每个不经意的决定，每次环境潜移默化的影响，最终都成就了个体本身。生命看似是一条线，其实是万万条路中的一条，而这路上你独享的风景就是密符的重要来源。其次，密码应当具备良好的易记性。其实设计密符就跟写诗一样，一句好的诗只消一瞥就永生难忘，一个好的密符亦是如此，设计完一看就能记住应当是好密符的重要特点。

接下来本文就以密符的设计为主题，从几个方面进行探讨:

密符的类型

一般而言，人们通常会遇到如下几类密符

  1. 6位数字的支付密码、开机密码（亦即PIN）
  2. 8位字符以上的字母、数字、符号组合密码（不做细分）
  3. 9X9的手势密码
  4. 指纹、人脸、瞳孔、声音等生物密码
  5. 其他各种花哨的密码（比如黑莓android系列内置的图案移位密码，好像Win10上曾经也有过）

第一类可以归为第二类，第四类由于其绝对的私人性，可以说几乎不存在泄露、遗忘的风险，在此笔者将着重讨论第二、三类密符的设计。首先笔者将从设计方法和应用2个方面对各类密符的设计进行探讨

为了实现密符的强私人性，我们需要一个专有的信息来源。这个来源必须是与我们生活直接相关的，如此才能保证我们可以更快更持久地记住这个密码，也自然的具有更高的强度。具体如下:

地址

地址类密符可以从地区的简称、地理位置(相对方位、地标景点等)来进行映射，接下来以笔者的家乡大美南阳为例:

但这样的映射还太简单，为了更具有私人性，再加上另一座笔者待过的城市乌鲁木齐来提高一下密符的强度地址密符：Sion@YuWan2XinU，含义是：我Sion从南阳到新疆(Xin)乌鲁木齐(Urumqi)

应用体系

在了解了诸多的来源之后，我们便可以一展宏图了。我们可以根据密码的应用场景来设计密码。本文推荐的密码应用体系包含四类密码，它们的强度与应用性各有差异：

  • 用于关键平台（比如微信、QQ、学信网等主要平台）
  • 用于金钱相关的需要特别加密的平台（支付密码）
  • 用一般社交兴趣网站（哔哩哔哩、知乎、微博等）
  • 用于杂毛网站（用后即弃的，很少再用的）

管理

再好的密符，再好的应用体系，也绝非灵丹妙药，难以保证长久的安全，在此笔者对于个人密符的管理提出来一些建议

老生常谈

  1. 定期更换
  2. 不滥用
  3. 使用密码管理软件
  4. 使用密码生成器，搭配特定软件自动填写

密码保护

众所周知，密码学发展至今，已经出现了许多优质的加密方法。对称密码算法如DES,AES等，它们使用一个秘钥来进行加密，在秘钥安全的情况下，硬破解极为困难(废话)。还有现代最为广泛应用的非对称加密——公私钥体系，这类密码利用了数学上的陷门单向函数，有RSA，DHM(区块链上有很多应用)等。以及从公钥体系发展来的数字签名算法使用哈希函数来进行防篡改验证，算法有MD5, SHA系列,DIMON等。最后还有为了抵抗量子计算机而研制的抗量子加密算法，有Merkle(基于哈希)
、HFE(基于多变量)、NTRU系列(基于格，在2020年openssh 9.0将其作为默认加密算法)
。密码保护，顾名思义，就是用加密算法把设计好的密码进行加密，这样即便密码不小心泄露，别人也无法破解出来真正的密码。你甚至可以把加密好的密码，明文放到网络上，就以稍微落后的AES加密算法来讲，破解起来也要费好些功夫。当然，谁会对你没几个钱、没几个朋友的账户感兴趣呢？

下面以OpenSSL软件为基础，实操一下[1]

使用AES来加密文件

加密过程中会要求输入一个加密密码，重复输入两次即可完成对文件的加密。解密一个文件的时候会要求输入加密文件时设置的密码才能进行解密。

  1. 使用 AES-256方法来加密一个文件 (data.zip 为原始文件，back.zip 为加密之后的文件)
  1. 使用 openssl 解密一个文件 (back.zip 为加密的文件，data.zip 为解密之后的文件)
  1. 解释
  • enc：表示对文件进行对称加解密，
  • -e： 表示对一个文件进行加密操作
  • -d 表示对文件进行解密操作
  • -aes256： 表示使用 aes256 算法进行操作
  • -in：表示需要被加解密的文件
  • -out： 表示加解密之后生成的新文件。

使用RSA来加密文件

  1. 首先需要使用 openssl 生成一个 2048 位的密钥 rsa.key 文件 (rsa.key 密钥文件中包含了私钥和公钥)
  1. 然后从 rsa.key 密钥文件中提取出公钥 pub.key
  1. 使用 pub.key 公钥加密一个文件 (data.zip 为原始文件，back.zip 为加密之后的文件)
  1. 使用 rsa.key 私钥解密一个文件 (back.zip 为加密的文件，data.zip 为解密之后的文件)

Chat-GPT的建议

你是一名密码学领域的教授，请设计一些个人日常用的密码，这些密码应该有两个最基本特点:强私人性与易于记忆。

当设计个人日常用的密码时，建议采取以下措施以确保强私人性和易于记忆：

  1. 使用短语作为密码：选择一个容易记住的短语，例如 “我的狗叫做巴比”，并将其转换为一个密码，例如 “WdJzBbBb!”，其中 W 是 “我的” 的首字母，d 是 “狗” 的首字母，z 是 “叫做” 的中间字母，以此类推。

  2. 添加数字和符号：将密码中的字母替换为数字或符号，例如将 B 替换为 8 或 %，将 S 替换为 5 或 $。这样可以让密码更加复杂和难以猜测。

  3. 使用不同的密码：不要在多个帐户上使用相同的密码，因为如果某个网站发生数据泄漏，黑客可以使用您的密码尝试登录其他网站。

  4. 定期更改密码：定期更改密码可以减少黑客攻击的风险，并且可以确保您的密码对那些已经获得它的人无效。

  5. 使用密码管理器：使用密码管理器可以帮助您存储和管理所有的密码，使其易于查找，同时也可以生成更加安全的密码。

总之，一个好的密码应该是既强私人性又易于记忆的。通过使用这些建议，您可以创建一个更加难以破解的密码并保护您的个人信息。

类似于前两点的方法可以再多补充一些吗？

当设计个人日常用的密码时，除了使用短语和添加数字和符号外，还可以采取以下措施以确保强私人性和易于记忆：

  1. 字母大小写混合：在密码中混合使用大写字母和小写字母可以使密码更加复杂。例如，将 “我爱跑步” 转换为 “W@RunL0ver”。

  2. 隐藏密码信息：使用密码中第 n 个字符之后的字符作为真实密码，例如将 “我的生日是1985年3月25日” 转换为 “@9853m25d”。

  3. 密码长度至少为12个字符：较长的密码比较短的密码更加安全，因为它们更难猜测。建议密码长度至少为12个字符。

  4. 避免使用常见单词和短语：不要使用容易猜到的单词或短语（如 “password” 或 “123456”），因为这些密码很容易被黑客破解。

  5. 避免使用个人信息：不要在密码中使用您的姓名、生日或其他个人信息，因为这些信息可能很容易找到，并通过猜测密码访问您的帐户。

  6. 使用多种字符类型：使用数字、符号和字母的组合可以增加密码的安全性，例如将 “我喜欢吃巧克力” 转换为 “IL0v3@Ch0c!”。

总之，一个好的密码应该是既强私人性又易于记忆的。通过使用这些建议，您可以创建一个更加难以破解的密码并保护您的个人信息。
